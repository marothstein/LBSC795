Fast User Interaction / Fitt's Law: Temperature Conversion UI

Exercise Description:

Given an existing temperature conversion UI (pictured below), we were tasked with redesigning it for speed as determined by Fitt's Law. Our goal was to design an interface with the lowest possible average and maximum times to completion for the main task (converting temperature between two scales).

Redesign:

We strove to provide a redesign that minimized time to completion, but wanted to remain conscious of some things we found frustrating about the original interface. For this reason, we made sure to include multiple avenues of interaction that would lead the user to their goal, even if some were potentially less efficient than others. To accommodate an extremely fast and well-versed expert user, we strove to include full keyboard interaction and as much automated task handling as possible. For example, a user can choose their destination temperature scheme by either clicking a button or typing the abbreviation letter for that scheme (F for Farenheit / C for Celsius). Along the same lines, the user could initiate the conversion by either clicking the "Convert" button or hitting the enter key.
